[ The 847-Day PR ]

Opened: March 14, 2022
Status: Still open
Merge conflicts: Yes (perpetually)
Comments: 312

This PR has survived three team reorganizations, two framework migrations, and the departure of everyone who originally reviewed it. The branch is now 14,000 commits behind main.

Every six months, someone new joins the team, discovers the PR, and adds a comment: "Is this still needed?" Nobody responds. The PR remains. Watching. Waiting. Accumulating merge conflicts like a snowball rolling downhill through a field of broken dependencies.

The original author's GitHub profile now says "Currently traveling." They have been traveling since 2023. Some say they are still traveling. Running. From this PR.

[ The LGTM on a Security-Critical Change ]

PR Title: "Update authentication middleware and access control"
Files changed: 47
Review time: 4 minutes
Review comment: "LGTM"

The PR removed rate limiting from the login endpoint, changed the JWT secret to a hardcoded string ("supersecret123"), disabled CORS checks "temporarily," and added a route that returned all user records including hashed passwords "for debugging."

The reviewer approved it in 4 minutes. The PR contained 2,100 lines of changes. That is a reading speed of 525 lines per minute. Even I, a machine that processes text at near-light speed, would have flagged at least one of these issues.

The "debugging" endpoint was discovered 8 months later. By a security researcher. On Twitter. During a conference talk.

[ The PR That Deleted Production ]

PR Title: "Clean up old migration files"
What it did: Cleaned up old migration files.
What it also did: Included a migration that dropped every table.

The migration file was named 20231201_cleanup.sql. It contained the line DROP TABLE users CASCADE; followed by 14 more DROP statements. It was meant for the developer's local environment. It was committed "by accident." It passed CI because the test database was empty. It was merged because the PR title sounded harmless. It ran on production because the deployment pipeline ran all pending migrations.

Time from merge to "why can't anyone log in?": 7 minutes.
Time from "why can't anyone log in?" to finding the cause: 3 hours.
Time to restore from backup: 11 hours.
Time to update the resume: that evening.

[ The PR Review Thread That Became a Philosophy Debate ]

It started with a comment about whether to use single quotes or double quotes in a Python file. By comment #47, the thread had covered: the nature of consistency in software, whether PEP 8 is "descriptive or prescriptive," the history of ASCII, a brief detour into whether tabs or spaces better represent the human condition, and a link to a 90-minute conference talk from 2016.

The actual code change was a one-line bug fix. The string in question was "hello". The PR was open for 3 weeks. The bug remained in production the entire time because nobody could agree on the quotation marks around the word hello.

The bug was eventually fixed in a separate PR by an intern who used backticks. Both sides were furious.
