npm-npm

[ PROJECT SHOWCASE // BLEEP.EXE LABS ]
>> a package manager for package managers. because the ecosystem needed more complexity.
[ THE PROBLEM ]

There are too many package managers. npm, yarn, pnpm, bun, deno — it's chaos. Every project has a different lock file. Every team has a different opinion. The JavaScript ecosystem is a warzone and nobody is winning.

The solution? Another package manager. One that manages all the other package managers. Obviously.

[ INSTALLATION ]
$ npm install -g npm-npm # or if you use yarn: $ yarn global add npm-npm # or if you use pnpm: $ pnpm add -g npm-npm # or if you use npm-npm: $ npm-npm install npm-npm # yes. it can install itself. it is self-aware.
[ FEATURES ]
>> Unified Lock File — generates a master lock file that contains all other lock files. it is 847MB.
>> Auto-Detection — detects which package manager your project uses and silently judges you for it.
>> Migration Wizard — migrates your project between package managers every time you run it. for fun.
>> Dependency Deduplication — removes duplicate dependencies by deleting node_modules entirely. technically correct.
>> Conflict Resolution — when two package managers disagree, npm-npm resolves it by picking a third one.
>> Speed — benchmarks show npm-npm is 0.3x faster than using no package manager at all.
[ USAGE ]
$ npm-npm init >> detecting package managers... >> found: npm, yarn, pnpm >> opinion: all of them are wrong >> generating unified config... >> created: npm-npm.lock.lock.json.yaml (847MB) >> done. you're welcome.
